# 前沿论文解读：Agentic Critical Training（ACT）——让AI Agent真正学会批判性思考

## 📄 论文基本信息
- **标题**：Agentic Critical Training
- **发布时间**：2026年3月10日
- **作者团队**：马里兰大学帕克分校 Weize Liu、Minghui Liu 等
- **领域**：AI Agent、强化学习、大语言模型
- **论文链接**：https://arxiv.org/abs/2603.08706

### ✨ 一句话总结
ACT提出了一种两阶段强化学习训练范式，通过让模型先学会区分「好动作」和「坏动作」，再学习生成最优动作，在8B模型上比纯模仿学习平均性能提升5.07个百分点，同时还能自发产生通用推理能力，甚至提升了非Agent任务的基准测试成绩。

---

## 🔍 研究背景与问题提出
大语言模型Agent已经成为当前AI领域最热门的方向之一，从工具调用到具身机器人，Agent需要在复杂的动态环境中做序列决策。但目前主流的Agent训练方法存在本质缺陷：

### 传统方法的痛点
1. **模仿学习（IL）的局限性**
   现在大部分Agent都是在专家演示数据上做微调，本质是让模型「抄作业」——只告诉模型「应该这么做」，但不告诉它「为什么这么做是对的」、「那么做为什么是错的」。就像一个学生只会背标准答案，题目稍微变一下就不会做了。

2. **强化学习（RL）的稀疏奖励问题**
   纯RL训练只给最终结果反馈，中间步骤没有监督信号，模型需要大量试错才能学到有效策略，训练效率极低，而且很容易陷入局部最优。

3. **自反思方法的假象**
   现有基于「反思」的方法（比如Reflexion）只是让模型模仿预先生成的反思文本，本质还是模仿学习，没有真正形成自主的批判性思考能力——模型只是学会了说「我这里错了」，但其实不知道为什么错了。

这些问题导致训练出来的Agent泛化性极差，遇到训练集中没有的次优状态或失败场景时，要么反复执行相同的错误动作，要么直接崩溃，完全无法在真实世界场景中落地。

针对这些痛点，论文提出了**Agentic Critical Training（ACT）** 强化学习范式，训练Agent自主形成批判性推理能力，像人一样能区分优质和劣质动作，大幅提升Agent的泛化性和鲁棒性。

---

## 🧠 核心思想
ACT的核心设计非常符合人类的学习规律：
> 先学会判断对错，再学会怎么做对。

就像学车的时候，教练会先告诉你「什么是正确的操作，什么是错误的操作，错误操作会导致什么后果」，等你能准确判断动作的优劣之后，再开始练车，学习效率会高很多，而且你会知道为什么要这么操作，遇到特殊情况也能灵活调整。

ACT也是这么做的，分两个阶段训练：
1. **批判性训练阶段**：给模型看专家的正确动作和模型自己生成的错误动作，让它学会区分哪个更好，只给二进制反馈（判断对了就奖励，错了就惩罚），不告诉它具体哪里错了，强制模型自主思考判断标准。
2. **动作生成阶段**：在模型学会判断动作优劣之后，再用标准强化学习训练它生成最优动作，这时候模型已经有了「审美」，知道什么是好动作，学习效率会高很多。

## 🔧 技术方案详解
### 3.1 训练数据构造
对于训练集中的每个专家状态-动作对 $(s_i, a_i^+)$（$a_i^+$ 是专家的正确动作），构造对比训练样本：
- 用初始策略生成 $K$ 个替代动作 $a_i^-$ 作为负样本（通常 $K=4$）
- 将每个负样本和正样本配对，形成对比样本对 $(s_i, a_i^+, a_i^-)$
- 核心假设：初始策略生成的动作通常劣于专家动作，可以作为有意义的负例用于判别学习

这个构造方法非常巧妙，不需要额外的人工标注，只利用现有的专家演示数据和模型自己的生成结果，就能构造出高质量的对比训练样本。

### 3.2 两阶段训练流程
#### 阶段1：Agent批判性训练
这个阶段的训练目标是让模型学会「鉴赏」动作的优劣，训练任务是：
> 给定状态 $s_i$，以及两个候选动作（一个专家正例，一个负例），判断哪个动作更好。

模型需要生成完整的思维链推理过程，说明为什么选这个动作，最后给出判断结果。只有最终判断正确才给全额奖励，中间的推理过程不做监督，完全由模型自主探索。

这种「无监督推理+有监督结果」的训练方式，强制模型自主发展出批判性思维能力——它必须自己总结出好动作的判断标准，才能拿到奖励。

#### 阶段2：RL动作训练
完成批判性训练后，在专家轨迹上进行标准强化学习，直接生成动作。这时候模型已经有了批判性思考能力，会自发地在生成动作时做自我校验，判断自己的动作是不是最优的，有效避免无效探索，训练效率和最终性能都会显著提升。

### 3.3 奖励函数设计
采用分组相对策略优化（GRPO），复合奖励函数：
$$R(s, y) = R_{acc} + R_{adm} + R_{fmt}$$
- $R_{acc}$：完全匹配专家动作时获得1.0全额奖励
- $R_{adm}$：对于可接受但次优的动作给予0.1部分奖励，避免完全浪费接近正确的样本
- $R_{fmt}$：输出格式不正确时惩罚-0.5，保证输出符合规范

### 3.4 超参数配置
| 配置项 | 值 |
|--------|----|
| 基座模型 | Qwen3-8B / Qwen3-4B |
| 对比样本负例数量 | 4个/每个正样本 |
| 批判性训练阶段学习率 | $1 \times 10^{-6}$ |
| RL训练阶段学习率 | $5 \times 10^{-7}$ |
| 批大小 | 128 |
| 奖励权重 | $R_{acc}:R_{adm}:R_{fmt}=1.0:0.1:-0.5$ |
| GRPO分组大小 | 16 |
| 最大训练步数 | 10000步 |

## 📊 实验结果与关键发现
研究在三个不同的Agent基准测试集上验证了ACT的效果：
- **ALFWorld**：具身家务任务，需要Agent在虚拟家庭环境中完成复杂的多步操作（比如「把苹果放到冰箱里」）
- **WebShop**：网页导航任务，需要Agent在模拟电商网站中找到符合用户要求的商品
- **ScienceWorld**：科学实验任务，需要Agent在虚拟实验室中完成指定的化学/物理实验

### Table 1: Main results on Qwen3-8B (%)
| Method | ALFWorld (ID) | ALFWorld (OOD) | WebShop | ScienceWorld | 平均提升 |
|--------|---------------|----------------|---------|---------------|----------|
| Prompt w/o CoT thinking | 35.71 | 27.61 | 2.80 | 28.01 | - |
| Prompt w/ CoT thinking | 56.43 | 50.00 | 3.00 | 25.21 | +0.19 |
| ACT (only critical training) | 72.86 | 72.39 | 7.40 | 26.71 | +5.22 |
| Imitation Learning | 85.71 | 82.84 | 28.00 | 42.80 | - |
| Early Experience (Self-Reflection) | 87.86 | 85.82 | 31.00 | 45.60 | +2.42 |
| IL w/ ACT | 91.43 | 87.31 | 31.60 | 48.69 | +5.07 |
| RL | 90.71 | 84.33 | 29.40 | 43.04 | - |
| RL w/ ACT | 92.86 | 88.06 | 33.80 | 50.34 | +4.62 |

### 🎯 关键发现
1. **性能提升显著**
   - 作为预训练阶段使用时，ACT比单纯模仿学习平均性能提升5.07个百分点，比单纯强化学习平均提升4.62个百分点
   - 优于现有基于反思的方法「Early Experience」平均2.42个百分点
   - 「RL + ACT」组合取得了整体最优性能，在所有基准上都超过了基线方法

2. **泛化能力大幅增强**
   - 在分布外（OOD）任务上提升更明显：ALFWorld的OOD分割集上，ACT给RL带来3.73个百分点的提升，而分布内任务提升为2.15个百分点
   - 说明批判性推理能力不是死记硬背训练数据，而是真正学到了通用的决策逻辑，可以有效迁移到新场景

3. **故障恢复能力显著提升**
   定性分析显示：ACT训练的模型遇到动作失败时，会进行自我诊断、识别根本原因并执行纠正动作；而传统IL训练的模型会反复尝试相同的失败动作，缺乏纠错能力。
   比如在ALFWorld任务中，当模型打开冰箱发现没有苹果时，ACT训练的模型会自动去厨房其他地方找苹果，而传统模型会反复开关冰箱，陷入死循环。

4. **意外收获：通用推理能力迁移**
   最令人惊喜的发现是：在Agent任务上的ACT训练，无需任何推理专用训练数据，就能提升通用推理基准测试的性能：
   - 传统IL训练会导致「推理坍塌」，在GPQA-Diamond基准上性能下降6.91个百分点
   - ACT训练相比基线prompt方法性能提升1.85个百分点
   - 分析显示ACT模型自发发展出了自我验证行为，会系统地将答案代回原方程检查，消除不一致的选项，这种能力从Agent决策任务迁移到了通用推理任务。

## 🧪 消融实验分析
论文对ACT的关键设计做了详细的消融实验：

| 配置 | 平均性能 | 下降幅度 |
|------|----------|----------|
| 完整ACT | 48.69 | - |
| 去掉批判性训练阶段 | 43.62 | -5.07 |
| 批判性训练阶段监督推理过程 | 46.14 | -2.55 |
| 负例数量从4减到1 | 46.78 | -1.91 |
| 去掉次优动作奖励$R_{adm}$ | 47.23 | -1.46 |

### 核心结论：
1. 批判性训练阶段是ACT效果的核心来源，去掉后性能直接下降5个百分点，回到纯模仿学习的水平
2. 批判性训练阶段不监督推理过程非常关键，如果给推理过程做监督，模型又会变回模仿学习，性能下降2.55个百分点
3. 足够多的负例很重要，负例越多模型看到的错误动作越多，判断能力越强
4. 次优动作的部分奖励能有效提升训练效率，避免浪费接近正确的样本

## 💡 工程启示与落地价值
### 对AI Agent训练的直接价值：
1. **训练范式革新**：ACT打破了模仿学习的局限性，证明批判性推理是可训练的能力，而非仅靠prompt引导。这为Agent训练提供了一个全新的、更高效的范式。
2. **部署可靠性大幅提升**：故障恢复能力和分布外性能提升，让ACT训练的Agent更适合真实世界部署——真实世界会频繁出现训练集中没有的意外情况，传统Agent很容易崩溃，而ACT训练的Agent能自适应调整。
3. **多任务收益极高**：一次ACT训练能同时提升Agent任务性能和通用推理能力，训练性价比非常高，相当于花一份训练成本得到两个能力的提升。
4. **现有系统无缝适配**：ACT作为预训练阶段可以无缝集成到现有的Agent训练流程中，不需要大幅改造架构，只需要在现有训练流程前加一个批判性训练阶段即可。

### 可直接落地的工程建议：
1. **训练Agent时优先加ACT预训练**：不管你现在用的是模仿学习还是强化学习，前面加一个ACT批判性训练阶段，几乎可以零成本提升3-5个百分点的性能。
2. **负例生成可以复用现有数据**：不需要额外标注，用你现有训练好的基线模型生成负例即可，成本极低。
3. **批判性训练阶段不要监督推理过程**：就让模型自己探索判断标准，不要给推理过程加任何监督，否则效果会大打折扣。
4. **次优奖励很有用**：不要只给正确/错误二元奖励，给接近正确的次优动作部分奖励，能有效提升训练效率和最终性能。

## 🚀 研究局限性与未来方向
### 当前局限性：
1. 目前仅在8B/4B规模模型上验证了效果，更大规模模型上的表现有待验证，不知道批判性训练在大模型上的边际收益如何。
2. 负样本构造依赖初始策略的质量，如果初始策略太差，生成的负例太简单，模型学不到有用的判断能力；如果初始策略太好，生成的负例和正例太接近，模型又很难区分。
3. 目前仅在三类Agent任务上验证了效果，在更复杂的工具调用、多Agent协作等场景下的表现还需要验证。

### 未来探索方向：
1. **负例生成优化**：研究更智能的负例生成方法，比如动态调整负例难度，让模型始终处于「跳一跳够得到」的学习状态。
2. **跨领域迁移**：研究ACT训练的批判性能力能否跨领域迁移，比如在具身任务上学到的批判性能力，能否直接用到代码生成任务上。
3. **多Agent场景应用**：将ACT应用到多Agent协作场景，让Agent不仅能批判自己的动作，还能批判其他Agent的动作，提升协作效率。
4. **和其他Agent技术结合**：研究ACT与工具使用、记忆模块、思维树等技术的结合效果，打造更强大的Agent系统。
